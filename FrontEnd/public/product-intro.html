01 / WHY KNOW-HOW

知识很多，
但“真正怎么做”
依然稀缺。

概念可以搜索，经验却往往散落在人的判断和细节里。

面对一个具体问题，人们需要的不是更多定义，而是谁遇到过类似处境、当时怎样判断、按什么顺序行动、哪里最容易失败，以及如何知道结果真的有效。

Know-how Hub 把这些隐性的实践经验转化为可理解、可执行、可验证、可复用、可持续迭代的结构化 Know-how 包。它既服务人的实践，也成为 AI 可以直接利用的专业上下文。

03 / DETAIL技巧、误区与专业细节

02 / THE LOOP

一个问题，怎样变成可复用的答案？

每一次解决新问题，都在为公共 Know-how 增加内容、证据或新版本。

STEP 01问

描述真实问题

任务 Agent 通过对话澄清目标、背景、限制与期望结果。

STEP 02找

匹配已有经验

搜索相关 Know-how，判断满足程度并识别信息缺口。

STEP 03集

多人密封贡献

实践者通过访谈、撰写或上传内容，分别提供真实经验。

STEP 04萃

评估、冲突与溯源

萃取 Agent 保留不同方案及适用边界，标记每项来源。

STEP 05长

交付并形成版本

自动分配积分、生成新版本，继续服务更多相似问题。

03 / AI NATIVE CORE

三类 Agent，围绕一个结果协作：
把事情真正做成。

AGENT / 01

任务 Agent

站在需求方身边，把模糊困扰变成可以被回答的好问题。

  • 澄清真实目标和约束
  • 检索已有 Know-how
  • 识别缺口并生成任务
  • 准备后续访谈提纲
AGENT / 02

对话 Agent

像一位有准备的访谈者，帮助实践者说出写作难以表达的细节。

  • 从真实案例开始追问
  • 捕捉判断依据和例外
  • 支持文档上传和续聊
  • 生成可编辑的结构化贡献
AGENT / 03

萃取 Agent

不追求把所有观点压成一个答案，而是呈现共识、差异与边界。

  • 评估多份独立贡献
  • 识别冲突和信息缺口
  • 保留来源与采纳关系
  • 按贡献程度自动分配积分
