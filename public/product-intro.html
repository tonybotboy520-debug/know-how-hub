01 / WHY 360 ZHIHUI

知识库记录“是什么”，
Know-how 回答
“怎样真正做成”。

通用大模型掌握大量公开知识，但很难稳定获得特定行业与场景中的实践判断、执行方法和取舍逻辑。

高价值经验往往存在于专家大脑、个人文档、会议与日常业务中。传统知识管理依赖人主动写文档，容易留下结论，却遗漏真正决定结果的判断依据、适用边界、例外处理与校验标准。

360智汇不是再建一个静态知识库，而是由真实问题触发经验提炼，允许多人独立贡献，再由 AI 完成访谈、比较、冲突处理、萃取和归因，让专业能力持续进入市场、被调用并形成新版本。

02 / WHAT IS TRADED

平台交易的不是一份文档，
而是一项能够持续工作的 Know-how。

01

专家 Know-how 市场

用户可以搜索和购买已有 Know-how；没有合适内容时，再发布新建或迭代悬赏。

02

公开阅读简版

公开展示问题、场景、预期结果、概括思路、必要输入、样例、贡献者、版本和价格，帮助用户判断是否适用。

03

AI Skill 完整版

完整保留详细流程、判断规则、边界、例外和校验逻辑，面向 AI 执行，不公开、不展示、不提供下载。

04

服务端受控执行

购买者获得期限内的远程调用权。平台加载完整能力并只返回业务结果，保护 Know-how 与企业数据。

03 / THE GROWTH LOOP

从真实需求，到可调用能力。

需求、专家贡献、AI 萃取、远程调用和版本迭代共同组成一个持续增长的闭环。

STEP 01搜

发现已有能力

搜索和语义匹配已有 Know-how，展示匹配程度、适用场景与推荐理由。

STEP 02问

澄清并发布悬赏

现有内容不足时，任务 Agent 澄清目标和约束，生成新建或迭代任务草稿。

STEP 03访

提取隐性经验

多位实践者分别通过 AI 访谈、文档导入或直接撰写提交密封贡献。

STEP 04萃

评估、冲突与归因

萃取 Agent 识别共识、差异、边界和缺口，并记录被采纳内容的贡献者。

STEP 05用

调用并持续迭代

生成完整能力与阅读简版，自动结算积分；后续任务继续产生新版本。

04 / AI NATIVE CORE

三类 Agent 分工协作，
让隐性经验真正进入产品。

AGENT / 01

任务 Agent

站在需求方一侧，把模糊困扰变成可以被搜索、评估和回答的专业任务。

  • 理解目标、背景与约束
  • 检索已有 Know-how
  • 自动识别信息缺口
  • 生成任务草稿与访谈提纲
AGENT / 02

对话 Agent

站在贡献者一侧，通过有准备的追问，把写作难以表达的判断和细节提取出来。

  • 从真实案例开始追问
  • 捕捉依据、边界和例外
  • 支持文档上传和中途续聊
  • 生成可编辑的结构化贡献
AGENT / 03

萃取 Agent

站在最终交付一侧，不强行消除分歧，而是呈现共识、不同方案与适用条件。

  • 评估多份独立贡献
  • 识别冲突和信息缺口
  • 生成完整 Know-how 与简版
  • 完成贡献归因与积分分配

借助 AI 访谈降低表达门槛，把隐性经验转化为署名、领域声誉和可持续收益的能力资产。

03

AI / Agent 开发者

通过远程接口，为 Agent 和商业产品接入结构完整、版本可控、持续迭代的专业能力。

04

企业与团队

购买外部专家能力，也把内部经验沉淀为组织资产，减少知识流失、重复探索与专业能力缺口。

06 / MARKET ENGINE

让专业经验获得长期回报。

积分统一连接悬赏、使用权、有效执行和贡献分配，让 Know-how 不只被生产，也愿意被长期维护。

MARKET 01充

购买平台积分

企业和 OPC 使用真实货币购买积分，进入平台后统一使用积分交易。

MARKET 02买

获得使用权

阅读简版免费；企业按月或按年购买使用权，OPC 可按次或购买小额调用包。

MARKET 03悬

生产新能力

没有合适 Know-how 时，用积分发布新建或迭代悬赏，吸引多人贡献。

MARKET 04执

按有效执行计费

企业套餐包含执行额度，超额后继续消耗积分；失败、超时和平台重试原则上不收费。

MARKET 05分

贡献者持续收益

悬赏和使用权按规则分配；执行积分先覆盖成本，再让贡献者获得持续调用收益。

07 / FROM PILOT TO ECOSYSTEM

先验证真实价值，
再形成公开市场与专业能力生态。

第一阶段的判断标准很直接：非专家能否利用 Know-how，完成原本需要专家能力的专业项目，或获得专业水平的结果。

360智汇先从公司内部真实需求出发，沉淀高价值种子内容并验证“任务—贡献—萃取—调用”闭环；随后向企业和 OPC 销售已有 Know-how 使用权，并承接新的悬赏需求。

当更多问题、专家、企业和 AI 产品进入平台，每一次调用和迭代都会增加内容、证据与版本，最终形成可被不同 Agent 和商业产品复用的专业能力基础设施。
